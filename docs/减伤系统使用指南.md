# TaczCurios 减伤/防护系统使用指南

> 三套系统均采用 **受击触发式**（参考 GoetyRevelation / RevelationFix 的 Apostle 减伤模式）。
> 饰品类在 `LivingHurtEvent` 中调用一次即可，无需在 `curioTick` 每 tick 维护。

---

## 一、吸收效果（低血量触发黄心）

**文件**: [CurioAbsorptionEventHandler.java](file:///d:/TaczAttributeAdd/TaczCurios%201.20.1/TaczCurios/src/main/java/com/xlxyvergil/tcc/event/CurioAbsorptionEventHandler.java)

**机制**: 受击时检查血量 + 装备 + 冷却，满足条件则赋予 `ABSORPTION`（黄心）效果并进入冷却。
冷却由 `onLivingTick` 自动倒计时，无需手动管理。

**配置** (`config/tcc-common.toml`):
```toml
[curio_absorption]
triggerHp = 0.25          # 触发血量阈值（0~1，默认 25%）
absorptionLevel = 4       # 效果等级（默认 ABSORPTION IV）
absorptionDuration = 60.0 # 效果持续（秒）
cooldown = 180.0          # 冷却（秒）
```

**饰品侧调用** — 在 `LivingHurtEvent` 中调用一行:

```java
@SubscribeEvent
public static void onLivingHurt(LivingHurtEvent event) {
    LivingEntity entity = event.getEntity();
    
    // 方式一：使用通用配置
    CurioAbsorptionEventHandler.tryTriggerAbsorption(
        entity,
        TccItems.YOUR_ITEM.get(),
        TaczCuriosConfig.COMMON.curioAbsorptionTriggerHp.get(),
        TaczCuriosConfig.COMMON.curioAbsorptionLevel.get(),
        TaczCuriosConfig.COMMON.curioAbsorptionDuration.get(),
        TaczCuriosConfig.COMMON.curioAbsorptionCooldown.get()
    );

    // 方式二：自定义参数（不依赖配置）
    // CurioAbsorptionEventHandler.tryTriggerAbsorption(
    //     entity, TccItems.YOUR_ITEM.get(), 0.3, 2, 30.0, 120.0);
}
```

**返回值**: `true` = 成功触发，`false` = 未满足条件（未装备/冷却中/血量不够）。

**辅助方法**:
```java
CurioAbsorptionEventHandler.resetCooldown(entity);   // 重置冷却
int ticks = CurioAbsorptionEventHandler.getCooldown(entity); // 查询剩余冷却
```

**冷却存储**: 玩家 `persistentData`（卸下饰品后冷却依然保留直至归零）。

## 二、适应效果（同类型伤害递减）

**文件**: [AdaptationEventHandler.java](file:///d:/TaczAttributeAdd/TaczCurios%201.20.1/TaczCurios/src/main/java/com/xlxyvergil/tcc/event/AdaptationEventHandler.java) + [CurioAdaptationCapability.java](file:///d:/TaczAttributeAdd/TaczCurios%201.20.1/TaczCurios/src/main/java/com/xlxyvergil/tcc/capability/CurioAdaptationCapability.java)

**机制**: 对重复类型的伤害逐步减免。每次受到相同类型伤害时，按 `adaptFactor^(受到次数-1)` 递减。

### 核心参数

| 参数 | 含义 | 示例 |
|------|------|------|
| `maxSlots` | 最多记忆的伤害类型数量 | 3 → 只适应最近3种 |
| `adaptFactor` | 每次适应的乘算因子 | 0.8 → 第一次 100%，第二次 80%，第三次 64% |
| `decaySeconds` | 无伤害 N 秒后清空记忆 | 30 → 30秒不受伤则重置 |

### 饰品侧调用

**装备时注册**:
```java
@Override
public void onEquip(SlotContext slotContext, ItemStack prevStack, ItemStack stack) {
    LivingEntity entity = slotContext.entity();
    entity.getCapability(CurioAdaptationCapability.CAPABILITY).ifPresent(handler -> {
        handler.register(
            "your_curio_id",  // 唯一标识，卸下时用同一个 id 注销
            3,                // maxSlots: 记忆 3 种伤害类型
            0.8,              // adaptFactor: 每次 ×0.8
            30                // decaySeconds: 30秒无伤重置
        );
    });
}
```

**卸下时注销**:
```java
@Override
public void onUnequip(SlotContext slotContext, ItemStack newStack, ItemStack stack) {
    LivingEntity entity = slotContext.entity();
    entity.getCapability(CurioAdaptationCapability.CAPABILITY).ifPresent(handler -> {
        handler.unregister("your_curio_id");
    });
}
```

### 伤害类型对照

`DamageSource.getMsgId()` 的常见值:

| msgId | 含义 |
|-------|------|
| `mob` | 近战攻击 |
| `player` | 玩家近战 |
| `arrow` | 弓箭 |
| `explosion.player` | 爆炸 |
| `magic` | 魔法伤害 |
| `inFire` / `onFire` | 火焰 |
| `drown` | 溺水 |
| `fall` | 摔落 |
| `tacz:bullet` | TACZ 枪械伤害 |
| `tcc:imaginary_damage` | 虚数伤害 |

### 衰减公式

```
第1次受到伤害类型X → 100% 伤害 → 记忆X
第2次受到伤害类型X → 100% × adaptFactor 伤害
第3次受到伤害类型X → 100% × adaptFactor² 伤害
...
```

当记忆槽满时，最久未受伤的类型被淘汰（LRU）。

## 三、受伤冷却 / 受伤上限（底层拦截）

**文件**: [DamageResistanceMixin.java](file:///d:/TaczAttributeAdd/TaczCurios%201.20.1/TaczCurios/src/main/java/com/xlxyvergil/tcc/mixin/DamageResistanceMixin.java)

**机制**: 在 `LivingEntity.setHealth()` 层面拦截血量下降，比任何事件层都更底层，不会被其他模组绕过。

提供两个独立机制，采用**受击触发式** — 在 `LivingHurtEvent` 中调用一次即可：

### 3a. 受伤冷却 — 受击后 N tick 内伤害全部归零

```java
@SubscribeEvent
public static void onLivingHurt(LivingHurtEvent event) {
    LivingEntity entity = event.getEntity();
    // 受击后 30 tick (1.5秒) 内伤害归零
    if (CurioSearchHelper.hasCurioEquipped(entity, TccItems.YOUR_ITEM)) {
        DamageResistanceHelper.setDamageCooldown(entity, 30);
    }
}
```

冷却由 `tick()` Mixin 自动递减，归零后自动移除，无需手动清理。

### 3b. 单次受伤上限 — 限制每次受伤的最大值

```java
@SubscribeEvent
public static void onLivingHurt(LivingHurtEvent event) {
    LivingEntity entity = event.getEntity();
    // 单次受伤不超过 20.0（10颗心）
    if (CurioSearchHelper.hasCurioEquipped(entity, TccItems.YOUR_ITEM)) {
        DamageResistanceHelper.setDamageCap(entity, 20);
    }
}
```

> 限伤需要持续维持（每次 hurt 刷新），或在 `curioTick` 中每 tick 设置。卸下时调用 `clearDamageCap(entity)` 清除。

### 完整 API

| 方法 | 作用 |
|------|------|
| `setDamageCooldown(entity, ticks)` | 受击触发冷却，N tick 内所有伤害归零 |
| `clearDamageCooldown(entity)` | 移除冷却（通常无需调用，自动归零） |
| `setDamageCap(entity, maxDamage)` | 设置单次受伤上限 |
| `clearDamageCap(entity)` | 移除上限（卸下饰品时调用） |
| `clearAll(entity)` | 一次性移除冷却+上限 |

### 优先级规则

冷却 > 上限。冷却生效时伤害直接归零，上限不参与计算。两者可同时设置，冷却结束后上限继续生效。

**注意**: 只拦截受伤（`delta < 0`），不拦截回血。对任意 LivingEntity 生效。

## 场景选择指南

| 需求 | 推荐方案 | 原因 |
|------|---------|------|
| 残血救命 | 方案一（吸收） | 黄心直观，有冷却不滥用 |
| 站撸 Boss 叠减伤 | 方案二（适应） | 随战斗时间增长自然递减 |
| 无敌帧 / 伪无敌 | 方案三a（冷却） | 最底层拦截，绝对免伤 |
| 防秒杀 | 方案三b（上限） | 限制单次最大受伤 |
| 残血免死 + 防秒杀 | 方案一 + 方案三b | 低血给黄心，同时限制单次伤害 |
