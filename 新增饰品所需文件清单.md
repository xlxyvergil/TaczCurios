# 新增饰品所需文件清单

## 一、Java 代码（4~6 个文件）

### 1. 饰品实现类
**路径**: `src/main/java/com/xlxyvergil/tcc/items/NewItem.java`

- 继承 `BaseCurioItem`（已封装 `onEquip`/`onUnequip` 自动调用效果 + TACZ 属性刷新）
- **必须实现**的方法：
  - `applyEffects(LivingEntity)` — 装备时应用属性修饰符
  - `removeEffects(LivingEntity)` — 卸下时移除属性修饰符
- 属性修饰符 UUID 用 `UUID.fromString()` 标准格式，不可随意编造
- 属性修饰符 name 用翻译键格式如 `tcc.new_item.xxx`
- **tooltip 全部使用语言键**（`Component.translatable`），禁止硬编码
- 如有切换武器逻辑，覆写 `applyGunSwitchEffect`（通常调用 `applyEffects`）
- 如需限制装备条件（如绑定），覆写 `canEquip`
- 如需禁止附魔，覆写 `isEnchantable` 返回 false

**进化链饰品额外需要**（参考 SummerBeach/BrahmaBeasts）：
- NBT 常量：`KILL_COUNTS_TAG`、`CARRIED_RESISTANCE_TAG`
- 静态方法：
  - `getResistanceFromKills(LivingEntity)` — 按抵抗实体列表加权计算
  - `getCarriedResistance(LivingEntity)` — 读取继承抗性
  - `incrementEntityKillCount(Player, String, int)` — 增加击杀计数并刷新属性
  - `areAllRequirementsMet(LivingEntity)` — 检查进化条件
  - `hasXxxEquipped(LivingEntity)` — 判断玩家是否装备
- Tooltip 需显示：抗性构成（继承+击杀）、进化需求列表、抵抗实体详情、绑定信息、槽位、稀有度、获取方式
- 构造函数中用 `.fireResistant()` 防火焰销毁

### 2. 物品注册
**路径**: `src/main/java/com/xlxyvergil/tcc/registries/TaczItems.java`

```java
import com.xlxyvergil.tcc.items.NewItem;

public static final RegistryObject<Item> NEW_ITEM = ITEMS.register("new_item",
    () -> new NewItem(new Item.Properties()
        .stacksTo(1)
        .rarity(Rarity.RARE)));  // 稀有度与文档一致
```

- `.stacksTo(1)` — 饰品必须最大堆叠 1
- `.rarity(Rarity.XXX)` — `COMMON`/`UNCOMMON`/`RARE`/`EPIC`

### 3. 创造模式标签页
**路径**: `src/main/java/com/xlxyvergil/tcc/creativetab/TaczCreativeTab.java`

在 `displayItems` 中添加：
```java
output.accept(TaczItems.NEW_ITEM.get());
```

### 4. 配置文件（如效果值需可配置）
**路径**: `src/main/java/com/xlxyvergil/tcc/config/TaczCuriosConfig.java`

```java
// 普通饰品
public final ForgeConfigSpec.DoubleValue newItemSomeValue;
// 定义:
builder.comment("XXX饰品配置").push("newItem");
newItemSomeValue = builder
    .comment("某属性加成 (默认: 0.5)")
    .defineInRange("someValue", 0.5, -1, 100);
builder.pop();
```

**进化链饰品配置**（参考 summerBeach/brahmaBeasts）：
```java
// 天火伤害保留倍率
public final ForgeConfigSpec.DoubleValue xxxHeavenFireMultiplier;
// 进化需求列表 [[实体, 数量], ...]
public final ForgeConfigSpec.ConfigValue<? extends List<? extends List<String>>> xxxEvolutionRequirements;
// 抵抗实体列表 [[实体, 每只抗性], ...]
public final ForgeConfigSpec.ConfigValue<? extends List<? extends List<String>>> xxxResistanceEntities;
// 抗性上限
public final ForgeConfigSpec.IntValue xxxMaxResistance;
// 击杀来源抗性上限
public final ForgeConfigSpec.IntValue xxxMaxKillResistance;
// 获取来源实体（仅起点饰品）
public final ForgeConfigSpec.ConfigValue<String> xxxObtainEntity;
```

### 5. 网络同步（必须同步新增配置）
**路径**: `src/main/java/com/xlxyvergil/tcc/network/ConfigSyncPacket.java`

**接收端** (`handleOnClient`) 和 **发送端** (`fromServer`) 都需要添加新配置的读写：
- 简单值：`config.addProperty("key", ...)` / `config.has("key")` → `.set()`
- 嵌套列表（如 evolutionRequirements）需特殊处理：外层 JsonArray 套内层 JsonArray，每元素 `List.of(get(0), get(1))`

### 6. 事件监听器（进化链饰品专属）
**路径**: `src/main/java/com/xlxyvergil/tcc/events/XxxEvent.java`

```java
@Mod.EventBusSubscriber(modid = "tcc", bus = Mod.EventBusSubscriber.Bus.FORGE)
public class XxxEvent {

    // 检查进化条件 — Curios 槽位手动操作（onUnequip → setStackInSlot → onEquip）
}
```

### 9. 中文语言文件
**路径**: `src/main/resources/assets/tcc/lang/zh_cn.json`

**物品名称**（按 Rarity 加颜色代码）：

| 稀有度 | Rarity 枚举 | 颜色代码 | 示例 |
|--------|------------|---------|------|
| 常见 | `COMMON` | `§9` | `"§9物品名"` |
| 罕见 | `UNCOMMON` | `§b` | `"§b物品名"` |
| 稀有 | `RARE` | `§6` | `"§6物品名"` |
| 史诗 | `EPIC` | `§d` | `"§d物品名"` |
| 传说 | `LEGENDARY`(自定义) | `§f` | `"§f物品名"` |
| 裂隙 | `RIFT`(自定义) | `§5` | `"§5物品名"` |

**基础翻译键**：
```json
"item.tcc.new_item": "§6物品名称",

"item.tcc.new_item.tooltip": "物品背景描述",
```

## 三、数据标签 & 配方（4~6 个文件）

### 11. Curios 槽位标签
**路径**: 根据饰品所属槽位选择：

| 槽位 | 文件 |
|------|------|
| TCC饰品栏 | `data/curios/tags/items/tcc_slot.json` |
| 逐火之蛾 | `data/curios/tags/items/tcc_3rd.json` |

添加 `"tcc:new_item"` 到 values 数组。

- `tcc_slot` — 绝大多数饰品
- `tcc_3rd` — 进化链饰品（夏日沙滩、梵天百兽、救世）

### 12. Forge 创造模式标签
**路径**: `src/main/resources/data/forge/tags/items/creative_tabs/tcc.json`

添加 `"tcc:new_item"` 到 values 数组。

### 13. 稀有度标签（用于合成内融核心）
根据饰品稀有度，在对应文件中添加：

| 标签文件 | 稀有度 | 内融核心数量 |
|---------|--------|-------------|
| `data/tcc/tags/items/common_curios.json` | COMMON | 1 |
| `data/tcc/tags/items/uncommon_curios.json` | UNCOMMON | 3 |
| `data/tcc/tags/items/rare_curios.json` | RARE | 5 |
| `data/tcc/tags/items/epic_curios.json` | EPIC | 9 |
| `data/tcc/tags/items/legendary_curios.json` | LEGENDARY | 12 |
| `data/tcc/tags/items/rift_curios.json` | RIFT | (特殊处理) |

### 14. 内融核心合成配方（如缺少对应稀有度的配方）
